FFF Table

This table gives an overview of the file-based filter farm (FFF). There is one line per appliance.

Explanation of the columns:
BU The host name of the builder unit responsible for the appliance (excluding the domain name '.cms')
The hyperlink leads to the BU hyperdaq page.
BU-
Info		 rateEvent rate in kHz (average over last second)
thruThroughput in MB/s (average over last second).
sizeEvent size in kB (average over last second).
#eventsNumber of events built since the start of the run.
#evts in BUNumber of events in the BU. A number > 0 means that the BU is working on events.
priorityThe priority of the event requests. Requests from BUs with a lower priority number are handled first. The priority is calculated based on the RAM disk occupancy.
#req. sentNumber of outstanding event requests. A number > 0 means that the BU is able to accept new events.
#req. usedNumber of requests the BU has received some data for and is assembling events.
#req. blockedNumber of blocked resources. A number > 0 means that the BU is throttled down. If the number is equal to the available resources, the BU stopped requesting events. A resource can get blocked when
  • the RAM disk gets full
  • there are not enough FU cores available
  • the HLT is lagging behind in processing data
#FUs HLTNumber of filter-unit (FU) CPU available in the appliance for HLT.
#FUs crashedNumber of filter-unit (FU) CPUs which repeatedly crashed in the current run.
#FUs staleNumber of filter-unit (FU) CPUs with a stale NFS mount.
#FUs cloudNumber of filter-unit (FU) CPU allocated to the offline cloud.
RAM disk usageOccupancy of in percent of the total RAM disk size in GB on the BU. If the RAM disk becomes full, the BU throttles the requests for new events.
#filesNumber of files written since the beginning of the run.
#LS w/ filesNumber of lumi sections (LS) for which the BU wrote at least one file.
current LSThe latest lumi section (LS) the BU knows about.
#LS for HLTNumber lumi sections (LS) waiting to be processed by the filter units.
#LS out HLTNumber lumi sections (LS) queued on the output of the filter units. -1 indicates that the FUs have not yet produced and output for the current run.
b/w outThe output bandwidth in MB/s of the HLT summed over all FUs in this appliance.
